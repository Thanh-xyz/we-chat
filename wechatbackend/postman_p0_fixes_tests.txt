Postman tests for Phase P0 fixes

Prerequisites
- Set baseUrl, userAccessToken, adminAccessToken, superAdminAccessToken.
- Set groupConversationId, directConversationId, archivedConversationId, targetUserId, superAdminUserId.
- All requests use JSON body and Authorization: Bearer <token> unless public.

1. P0-1 Search group conversation
GET {{baseUrl}}/api/conversations/search?q=project&limit=20&offset=0
Authorization: Bearer {{userAccessToken}}
Expected:
- 200 OK
- Response is an array.
- No BadSqlGrammarException.
- No duplicate id in response.

Postman test script:
const ids = pm.response.json().map(item => item.id);
pm.test("search group returns 200", () => pm.response.to.have.status(200));
pm.test("no duplicate conversation", () => pm.expect(new Set(ids).size).to.eql(ids.length));

6. P0-2 Block SYSTEM message exact uppercase
POST {{baseUrl}}/api/conversations/{{groupConversationId}}/messages
Authorization: Bearer {{userAccessToken}}
Body:
{
  "messageType": "SYSTEM",
  "content": "fake system message"
}
Expected:
- 403 Forbidden
- message: System messages can only be created by backend workflow

Postman test script:
pm.test("SYSTEM is forbidden", () => pm.response.to.have.status(403));
pm.test("clear error message", () => pm.expect(pm.response.text()).to.include("System messages can only be created by backend workflow"));
